CBV，FBV

  • 开发模式

    • 普通开发方式(前后端放在一起写)
    • 前后端分离
      • 所有的交互都是ajax实现的
      • 为前端提供URL（api/接口的开发）
        注意：永远返回HttpResponse

    • Django FBV CBV

      • FBV function base view

      • CBV class base view

        • 只要是基于CBV的框架几乎都是使用反射实现的，也就是根据method不同，执行getattr获取到，然后执行相应的方法

视图

路由

  • 列表生成式
  • 面向对象
    • 封装
      • 类可以对同一类的方法进行封装
      • 将数据封装到对象中

CBV总结

  • 本质：基于反射来实现
  • 流程：路由，view，dispatch（反射）

  • 取消csrf认证（装饰器要加在dispatch方法上且method_decorator装饰）

  • 扩展

    • csrf
      • 基于中间件的process_view方法
      • 装饰器给单独函数进行设置（认证或无需认证）
